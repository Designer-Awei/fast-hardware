# Fast Hardware - 项目级隔离 PRD（6-project-isolation）

> **状态（2026-04）**：**已落地** — 右侧聊天、画布内存态与固件编辑器草稿均按 **项目（`projectId`）** 做**完整隔离**；默认 `未命名项目` 标签策略已与产品一致。下文保留原始需求表述与验收记录，便于对照。

## 1. 背景
- 当前右侧仅一个聊天会话容器。
- 产品已支持多项目（新建/打开项目 + 标签切换）；历史上聊天上下文未按项目隔离会导致跨项目串话、历史污染和操作意图混淆。
- 现已补齐「项目维度会话与相关状态隔离」能力，并统一默认启动态。

## 2. 目标
1. 每个项目标签对应独立会话历史（messages / trace / 技能进度态）。
2. 切换项目标签时，右侧聊天区同步切换到该项目会话。
3. 应用启动后默认即显示一个标签：`未命名项目`，作为默认会话与后续打开项目的区分锚点。

## 3. 范围
### In scope
- 会话主键从“全局单实例”升级为“projectId 维度”。
- 项目切换时聊天区 UI、输入框上下文、技能编排状态切换。
- 默认标签页展示逻辑调整（启动即有 `未命名项目` 标签）。

### Out of scope（本期不做）
- 会话跨设备同步。
- 多用户权限与共享协作会话。
- 复杂分支会话（同项目多线程聊天）。

## 4. 用户故事
1. 作为用户，我在项目 A 聊到一半切到项目 B，不希望 B 看到 A 的历史。
2. 作为用户，我回到项目 A 时希望恢复 A 的聊天上下文与最近进度。
3. 作为新用户，首次打开应用就看到 `未命名项目` 标签，知道当前会话归属。

## 5. 功能需求
### FR-1 会话数据结构
- 新增 `projectSessions: Record<projectId, SessionState>`（或等价 Map）。
- `SessionState` 至少包含：
  - `messages[]`
  - `currentUserMessage` / 输入中间态
  - `skillsFlow` 相关瞬时状态（如阶段标签、trace message id）
  - 可选：最近使用模型、会话元信息

### FR-2 标签切换联动
- 当项目标签切换事件触发：
  1) 保存当前会话到当前 `projectId`
  2) 加载目标 `projectId` 会话
  3) 重绘聊天区与相关状态
- 若目标项目首次进入，则初始化空会话（含欢迎语策略按现有产品规则）。

### FR-3 默认标签行为
- 应用启动后，无论是否已打开外部项目，顶部至少展示一个默认标签：`未命名项目`。
- 默认标签建议使用稳定 `projectId`（如 `default-unnamed`），避免每次启动生成新 id 导致无法恢复。

### FR-4 与 Agent Loop 关系
- `runSkillsAgentLoop` 与相关进度事件必须绑定当前 `projectId` 作用域。
- 项目切换时：
  - 若旧项目仍在编排中，需明确策略（推荐：提示并中断旧项目编排；或继续后台执行但不污染当前标签）。
  - 当前标签只显示本项目的 agent 进度与最终消息。

## 6. 非功能要求
- 切换标签耗时可感知 < 100ms（仅本地状态切换，不含网络请求）。
- 不允许跨项目消息串写。
- 在异常恢复后，至少保证消息数组不丢失且归属正确。

## 7. 数据与状态建议
- `projectId` 来源优先级：
  1) 已保存项目的唯一标识（路径或内部 id）
  2) 默认未命名项目固定 id
- 持久化（可选）：
  - 若要跨重启保留会话，可在后续版本落地本地存储（按 `projectId` 分桶）。

## 8. 验收标准（测试场景）
1. 启动应用：顶部显示 `未命名项目` 标签；聊天区为空或默认欢迎语。
2. 在 `未命名项目` 发 2 条消息；新建项目 A，A 会话不包含未命名项目消息。
3. 在 A 发 1 条消息；切回 `未命名项目`，仍显示原 2 条；切回 A，仍显示 1 条。
4. 项目 A 触发 agent loop 后切到 B，不应在 B 标签看到 A 的工具进度/回答。
5. 重发、编辑、导出等行为仅作用于当前 `projectId` 会话。

## 9. 实施建议（轻触）
1. 先做内存级隔离（不落盘），打通 UI 与编排状态。
2. 再做会话持久化（可选）。
3. 最后补并发策略（项目切换时的在途 agent 任务处理）。

---

## 10. 已落地状态（2026-04）

### 已完成
- 已实现“项目维度聊天会话隔离”（内存级）：标签切换时保存/恢复对应 `projectId` 的消息会话。
- 已实现默认标签策略：启动时若无打开项目，自动创建并展示 `未命名项目`；关闭最后一个标签时同样回落到 `未命名项目`。
- 已实现画布内存改动恢复：项目切换回流时，优先恢复内存中的 `canvasData`（含元件/连线与视图状态），防止未保存改动被磁盘初始状态覆盖。
- 已实现固件编辑器项目隔离：代码编辑器草稿状态按项目保存/恢复。
- 已修正未保存项目代码保存逻辑：在未落盘项目中保存代码仅做内存暂存，不再强制触发“保存项目”。

### 验收结论
- 用户手工复测通过：
  1) 项目间聊天会话隔离正常；
  2) 未保存画布改动跨标签切换可恢复；
  3) 未保存项目下固件代码“保存”为内存暂存，交互提示与实际行为一致。

### 后续可选项
- 会话与草稿跨重启持久化（按 `projectId` 分桶落盘）。
- 在途 Agent Loop 的并发与切换策略细化（后台继续/切换中断可配置）。
